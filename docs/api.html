BhadaSeva API Documentation

Detailed reference for every implemented route under src/app/api in the current codebase. This document is hand-authored from the active route handlers and validation schemas, so it reflects the API behavior as currently implemented rather than an idealized contract.

Generated from code review of Next.js App Router route files, validation schemas, and model usage. Responses are described at a high level because many handlers return raw Mongoose documents.

Auth Legend

  • Public: no session required.
  • Authenticated: any signed-in user session.
  • Owner-scoped: requires auth and is intended for owner operations or owner-owned resources.
  • Tenant-scoped: requires auth and returns tenant data for the current tenant profile.
  • Profile-dependent: requires auth and behavior changes based on the active owner/tenant profile.

Important Notes

  • Several POST and PATCH handlers accept request bodies and pass fields directly into Mongoose operations. The practical fields documented here are based on current models and route usage, but some handlers do not strictly whitelist fields.
  • Some routes only check for an authenticated session and rely on session profile fields such as ownerProfileId/tenantProfileId to scope data.
  • The NextAuth catch-all route powers sign-in/session flows and is not a typical business CRUD endpoint.

Authentication and Users

GET Public /api/auth/[...nextauth]

Delegates to NextAuth. Used for built-in auth/session flows such as fetching session-related state or auth pages handled by NextAuth.

Purpose

Framework-managed auth endpoint exposed by NextAuth(authOptions).

Path Params

  • [...nextauth]: catch-all segment consumed by NextAuth internals.

Query Params

Varies by NextAuth action and provider flow.

Request Body

None for normal GET requests.

Success

  • Returns NextAuth-managed responses for session/auth actions.

Errors

  • Depends on NextAuth action and auth state.
POST Public /api/auth/[...nextauth]

Delegates to NextAuth for sign-in, credential submission, callbacks, and related auth actions.

Purpose

Handles credential sign-in and other NextAuth POST actions.

Path Params

  • [...nextauth]: NextAuth action route segment.

Request Body

Varies by NextAuth action. For credentials auth, body ultimately includes email and password fields expected by the configured credentials provider.

Success

  • Returns NextAuth-managed sign-in/callback response.

Errors

  • Credential or callback failures surface through NextAuth error handling.

Notes

  • Configured auth provider is credentials-based using email and password.
POST Public /api/users

Create a new user account.

Request Body

  • name (string, required, min length 2)
  • email (string, required, valid email)
  • phone (string, optional)
  • password (string, required, min length 6)
  • confirmPassword (string, required, must match password)

Success

  • 201 with created user object excluding password.

Errors

  • 400 invalid input.
  • 409 email already exists.
  • 500 internal server error.

Notes

  • Email is lowercased before lookup/storage.
  • Password is hashed with bcrypt before persistence.
POST Authenticated /api/users/switch-profile

Switch the signed-in user’s active profile between owner and tenant.

Request Body

  • profileType (string, required): owner or tenant

Success

  • 200 with success message and the new activeProfileType.

Errors

  • 401 unauthorized.
  • 400 invalid profile type or profile not enabled.
  • 404 user not found.
  • 500 internal server error.

Notes

  • The requested profile type must already be present in enabledProfiles.

Profiles

POST Authenticated /api/profiles/owner

Create an owner profile for the current signed-in user.

Request Body

  • displayName (string, required)
  • address (string, required)
  • ward (string, optional)
  • district (enum, required): Kathmandu, Lalitpur, Bhaktapur
  • contactPhone (string, required)
  • contactEmail (string, required, valid email)
  • panNumber (string, optional)
  • bio (string, optional)

Success

  • 201 with created owner profile.

Errors

  • 401 unauthorized.
  • 400 validation error.
  • 409 owner profile already exists.
  • 500 internal server error.

Notes

  • Also updates the user with ownerProfileId, enables the owner profile, and sets activeProfileType to owner.
POST Authenticated /api/profiles/tenant

Create a tenant profile for the current signed-in user.

Request Body

  • displayName (string, required)
  • occupation (string, optional)
  • employerName (string, optional)
  • permanentAddress (string, optional)
  • emergencyContactName (string, optional)
  • emergencyContactPhone (string, optional)
  • citizenshipNumber (string, optional)

Success

  • 201 with created tenant profile.

Errors

  • 401 unauthorized.
  • 400 validation error.
  • 409 tenant profile already exists.
  • 500 internal server error.

Notes

  • Enables the tenant profile on the user record.
  • If the user has no active profile yet, this route sets activeProfileType to tenant.

POST Owner-scoped /api/properties/[id]/units/[unitId]/assign

Assign a tenant to a vacant unit and create a lease.

Path Params

  • id: property id.
  • unitId: unit id.

Request Body

  • tenantEmail (string, required)
  • monthlyRent (number, required)
  • depositAmount (number, required)
  • startDate (date/string, required)
  • rentDueDay (number, required, expected 1-28)

Success

  • 201 with the created lease.

Errors

  • 401 unauthorized.
  • 404 property or unit not found, or ownership mismatch.
  • 400 unit is not vacant, or tenant account/profile not found.
  • 500 internal server error.

Side Effects

  • Creates a Lease with status active.
  • Updates the unit to occupied and sets currentLeaseId.

Marketplace Listings

GET Public /api/listings

Search and filter active public marketplace listings.

Query Params

  • q (string, optional): case-insensitive title search.
  • type (string, optional): listing/property type.
  • district (string, optional): district filter.
  • minPrice (number-like string, optional): minimum rent amount.
  • maxPrice (number-like string, optional): maximum rent amount.
  • furnished (string, optional): only true activates furnished filtering.

Success

  • 200 with up to 50 active listings sorted by newest first.

Errors

  • 500 internal server error.

Notes

  • Always applies isActive: true.
GET Public /api/listings/[id]

Path Params

Success

  • 200 with listing document populated with owner contact fields.

Errors

  • 404 listing not found or inactive.
  • 500 internal server error.

Side Effects

  • Increments viewCount by 1 after fetching the listing.
